Johnny.Decimal 文件分类法

先设计地图，再放文件：区域 Area → 分类 Category → ID 内容容器
一、具体怎么分类（核心：区域 → 分类 → ID）

A1. 区域 Area：先分 10 个大柜子

C3. ID：真正存内容的文件夹

AC.ID= 分类号 + 小数点 + 两位内容号
15.22 Travel insurance 15.41 All short trips 15.52 Trip to NYC 13.14 Tax returns 14.03 Password manager export
ID 是具体对象 / 主题 / 事件。
一个 ID 可包含文件、邮件、笔记、实体物品引用。
文件只放进 ID，不放进 Area 或 Category。

E5. 好分类 vs 坏分类

好分类：稳定、可复用、能预测

  • 按领域：15 Travel
  • 按稳定主题：13 Money
  • 按可复用容器：15.41 All short trips
  • 按具体事件：15.52 Trip to NYC
  • 名称清晰，可长期使用、可扩展

坏分类：临时、模糊、依赖情绪

  • 按情绪：杂七杂八、想到再放
  • 按工具：电脑里的东西、手机里的文件
  • 按紧急度：马上处理、待办、紧急
  • 无限嵌套：项目/资料/最终版/新版
  • 名称含糊：资料、重要、备份、其他
记住：Johnny.Decimal 的关键不是“给文件夹编号”，而是先建立稳定地图，再把每个文件放进一个明确的 ID 内容容器。
二、结构、索引与长期维护

F文件结构示例（10-19 Life admin）

10-19 Life admin ├─ 13 Money │ ├─ 13.14 Tax returns │ └─ 13.31 Home insurance └─ 15 Travel ├─ 15.01 Inbox for category 15 ├─ 15.22 Travel insurance │ └─ 2024-08-08 NYC lost bag/ │ └─ Claim form.pdf ├─ 15.41 All short trips │ ├─ 2024-02-13 CBR-SYD Weekend in Sydney/ │ └─ 2026-01-07 Hotel in Cobar/ └─ 15.52 Trip to NYC

G编号怎么读：以 15.52 Trip to NYC 为例

15.52 Trip to NYC
1 → 10-19 区域组（Life admin）
15 → Travel 分类（旅行）
.52 → 第 52 个 ID（内容容器）
小数点前 = 结构；小数点后 = 内容编号

H标准零位 Standard zeros：管理用桶

零位名称用途使用层级
AC.00JDex / Index记录 ID、位置、说明、关键词系统 / 区域 / 分类
AC.01Inbox临时收件箱，不确定归属时先放这里系统 / 区域 / 分类
AC.02Task & project management任务、项目、行动清单系统 / 区域 / 分类
AC.03Templates模板、表单、清单、可复用文档系统 / 区域 / 分类
AC.04Links链接、书签、参考资料、外部资源系统 / 区域 / 分类
AC.05-AC.08Reserved保留，不使用—
AC.09Archive归档：不再活跃但需保留的资料系统 / 区域 / 分类
优先使用最具体的 zero：分类级 > 区域级 > 系统级。

I三条硬规则：长期保持系统清晰

1区域和分类只导航，不直接放文件
所有实际内容都必须装在 ID 文件夹里。
2ID 内最多一层子文件夹
日期用 yyyy-mm-dd，排在文件名最前。
3超过 10/10/100 时扩系统或重分区
保持简单、清晰、可维护。
